### 用户故事详情

#### US-01 实时视觉理解

**作为**一名用户，**我想**打开摄像头让 AI 看到我的画面，**以便** AI 能理解我所处的环境。

**验收标准：**
- 点击「开启摄像头」按钮后，视频预览区显示实时画面
- 发送文字消息时，自动截取当前帧作为图片一并发送给 AI
- AI 能准确描述画面中的物体、场景、文字
- 画面静止时不会重复发送相同帧（智能帧采样）

**实现方案：**
- 前端通过 `getUserMedia` 获取摄像头视频流
- 使用 Canvas 截取视频帧，缩放至 480×360，JPEG 质量 0.6
- MSE（均方误差）算法检测帧差异，阈值 15，静止帧跳过
- 图片以 base64 编码通过 WebSocket 发送至后端
- 后端使用 OpenAI 兼容格式调用各提供商的视觉模型

#### US-02 文字对话

**作为**一名用户，**我想**通过文字与 AI 对话，**以便**询问关于画面的问题。

**验收标准：**
- 输入框输入文字后点击发送或按回车，消息发送成功
- AI 回复显示在对话区，带头像和时间
- 支持多轮对话，AI 记住上下文（最多 20 轮）
- 对话内容支持清空

**实现方案：**
- 前端通过 WebSocket 发送 JSON 消息 `{"type": "chat", "text": "...", "image": "..."}`
- 后端 `ChatService` 维护消息历史，自动裁剪至 20 轮
- AI 回复经过 emoji 和装饰符号清理，适合语音朗读

#### US-03 语音输入

**作为**一名用户，**我想**按住说话让 AI 听到我的语音，**以便**不用打字也能交互。

**验收标准：**
- 按住「按住说话」按钮开始录音，松开结束
- 录音过程中有音量波形动画反馈
- 识别结果自动填入输入框
- 识别失败时有明确错误提示，首次失败自动重试一次

**实现方案：**
- 前端通过 `getUserMedia` 获取麦克风音频（16kHz PCM）
- 通过 WebSocket `/ws/stt` 实时发送 PCM 帧至后端
- 后端 `StreamTranscriber` 使用 VAD 检测语音起止
- 识别优先级：DashScope Recognition API → FunASR 本地 → Google
- STT 状态面板显示当前引擎、识别状态、置信度

#### US-04 TTS 语音播放

**作为**一名用户，**我想**让 AI 自动朗读回复，**以便**我听到语音反馈。

**验收标准：**
- AI 回复后自动朗读（需用户先与页面交互）
- 点击消息气泡中的喇叭图标可手动播放/暂停
- 朗读过程中图标变为暂停状态
- 支持中文语音，自动选择系统中的中文语音包

**实现方案：**
- 使用浏览器 Web Speech API（`speechSynthesis`）
- 添加 `_hasUserInteracted` 守卫，符合浏览器自动播放策略
- 清理文本中的 emoji、HTML 标签、装饰符号后再朗读
- 预加载语音列表，优先选择 `zh` 开头的语音

#### US-05 多模型切换

**作为**一名用户，**我想**切换不同的 AI 模型提供商，**以便**选择最适合我的服务。

**验收标准：**
- 设置面板支持 5 家提供商选择
- 输入 API Key 后自动验证有效性
- 切换后所有对话自动使用新提供商，无需重启
- 底部状态栏显示当前提供商信息

**实现方案：**
- 后端 `PROVIDER_CONFIG` 存储 5 家提供商的配置（base_url、模型名、价格）
- 前端设置面板通过 REST API `/api/config` 读写配置
- 切换时后端创建新的 `ChatService` 实例，广播 `provider_changed` 给所有 WebSocket 客户端
- 全局统计在切换时重置，避免不同提供商的价格混淆

#### US-06 自动观察模式

**作为**一名用户，**我想**开启自动观察模式，**以便** AI 持续关注画面变化。

**验收标准：**
- 打开开关后，每 2 秒自动采样画面
- 画面有变化时自动发送给 AI 分析
- 画面静止时显示「静止」状态，不消耗 API 调用
- 视频覆盖层显示当前采样状态

**实现方案：**
- `setInterval` 每 2 秒触发 `autoSampleTick()`
- MSE 差异超过阈值时，发送帧给 AI 并附带 prompt「请描述当前画面的变化」
- 视频覆盖层显示「采样中」/「检测到变化」/「静止」状态

#### US-07 快捷指令

**作为**一名用户，**我想**使用快捷指令，**以便**快速完成常见任务。

**验收标准：**
- 提供 OCR识别、翻译、识别物体、描述场景、截图 5 个快捷指令
- 点击后自动截取画面并发送对应的 prompt 给 AI
- 截图指令可保存当前帧为 PNG 文件

**实现方案：**
- 前端预定义 5 个 prompt 模板
- 点击 chip 按钮时截取画面 + 发送对应 prompt
- 截图功能使用 Canvas `toDataURL` + `<a>` 标签下载

#### US-08 费用统计

**作为**一名用户，**我想**查看 API 调用费用统计，**以便**控制使用成本。

**验收标准：**
- 底部状态栏实时显示 API 调用次数、Token 用量、预估费用
- 每次 AI 回复后数字立即更新
- 费用超过阈值时状态栏变红警告
- 切换提供商时统计重置

**实现方案：**
- 后端 `_global_stats` 维护全局统计
- `add_stats()` 在每次 AI 调用后直接累加（无 delta 机制）
- WebSocket 回复附带 `usage` 字段，前端立即更新
- 前端每 3 秒轮询 `/api/stats` 兜底刷新
- 使用相对路径 `/api/stats` 兼容任意部署端口

#### US-09 设置面板优化

**作为**一名用户，**我想**在设置面板中方便地管理 API Key，**以便**快速配置。

**验收标准：**
- 密钥输入框切换可见/不可见时无跳变
- 快捷链接（申请 API Key）样式统一，融入深色主题
- 切换可见性使用 SVG 图标，跨平台一致

**实现方案：**
- 用 `.api-key-input` class 选择器替代 `input[type="password"]`，消除类型切换跳变
- 快捷链接设计为 pill 按钮样式，hover 时 accent 色高亮
- 密钥可见性切换使用 SVG eye/eye-off 图标替代 emoji

#### US-10 移动端适配

**作为**一名用户，**我想**在移动端也能使用这个工具，**以便**随时随地使用。

**验收标准：**
- 768px 以下屏幕布局自动切换为纵向排列
- 视频区和对话区上下分布
- 设置弹窗从底部滑出
- 按钮和文字大小适配触屏操作

**实现方案：**
- CSS 媒体查询 `@media (max-width: 768px)` 切换布局
- 视频区固定高度 35vh，对话区自适应
- 设置弹窗 `position: fixed; bottom: 0` 底部弹出

### 采用的成本控制策略详情

#### CC-01 图片压缩

**策略：** 将摄像头帧缩放至 480×360，JPEG 质量 0.6。

**效果：**
- 原始 640×480 JPEG 0.8 ≈ 50~100KB
- 压缩后 480×360 JPEG 0.6 ≈ 20~40KB
- 数据量减少约 50~60%，API 处理时间相应缩短

**实现位置：** `app.js` 的 `captureFrame()` 函数

```javascript
const MAX_W = 480, MAX_H = 360;
let w = videoEl.videoWidth, h = videoEl.videoHeight;
if (w > MAX_W || h > MAX_H) {
  const scale = Math.min(MAX_W / w, MAX_H / h);
  w = Math.round(w * scale);
  h = Math.round(h * scale);
}
canvasEl.toDataURL("image/jpeg", 0.6);
```

#### CC-02 智能帧采样

**策略：** 使用 MSE（均方误差）算法比较相邻帧，差异低于阈值时跳过发送。

**效果：**
- 摄像头静止时（如用户不移动），完全不消耗 API 调用
- 典型场景下节省 50~80% 的视觉 API 调用

**实现位置：** `app.js` 的 `calcMSE()` 和 `captureFrameSmart()` 函数

```javascript
const FRAME_DIFF_THRESHOLD = 15;
// 每 16 个像素采样一次（性能优化）
for (let i = 0; i < px1.length; i += 16) { ... }
```

#### CC-03 多模型分级

**策略：** 聊天使用便宜的 `qwen-turbo`，视觉识别使用专用的 `qwen-vl-max`。

**效果：**
- `qwen-turbo`：输入 0.3 元/百万 Token，输出 0.6 元/百万 Token
- `qwen-vl-max`：价格更高，但仅在有图片时使用
- 纯文字对话走便宜模型，大幅降低日常使用成本

**实现位置：** `ai_service.py` 的 `PROVIDER_CONFIG` 和 `chat()` / `chat_with_image()` 方法

```python
"qwen": {
    "chat_model": "qwen-turbo",      # 纯文字用便宜模型
    "vision_model": "qwen-vl-max",   # 视觉用专用模型
    "input_price": 0.3,
    "output_price": 0.6,
}
```

#### CC-04 Token 上限控制

**策略：** 设置 `max_tokens=512`，限制 AI 单次回复的最大长度。

**效果：**
- 避免 AI 生成过长回复，节省输出 Token 费用
- 512 Token 约 250 个中文字，足够简短描述和回答

**实现位置：** `ai_service.py` 的 `PROVIDER_CONFIG["qwen"]["max_tokens"]`

#### CC-05 对话历史裁剪

**策略：** 对话历史超过 20 轮时，保留系统提示 + 最近 20 条消息。

**效果：**
- 控制每次 API 调用的输入 Token 数量
- 避免历史过长导致的 Token 爆炸
- 20 轮约消耗 2000~4000 Token，可控范围内

**实现位置：** `ai_service.py` 的 `_trim_history()` 方法

```python
MAX_HISTORY = 20
def _trim_history(self):
    if len(self.messages) > MAX_HISTORY + 1:
        self.messages = [self.messages[0]] + self.messages[-(MAX_HISTORY):]
```

#### CC-06 超时保护

**策略：** 视觉识别 20 秒硬超时，文字对话 30 秒硬超时。

**效果：**
- 避免 API 无响应时用户无限等待
- 超时后返回友好提示，不消耗后续资源
- 使用 `asyncio.wait_for` + `run_in_executor` 不阻塞事件循环

**实现位置：** `main.py` 的 WebSocket 消息处理

```python
reply = await asyncio.wait_for(
    loop.run_in_executor(None, ai_service.chat_with_image, text, image),
    timeout=20,
)
```

#### CC-07 自动观察防抖

**策略：** 自动观察模式下，画面静止时显示「静止」状态，不发送 API 请求。

**效果：**
- 用户开启自动观察后，大部分时间画面是静止的
- 仅在检测到变化时才调用 API，节省 80%+ 的自动观察成本

**实现位置：** `app.js` 的 `autoSampleTick()` 函数

#### CC-08 费用实时监控

**策略：** 底部状态栏显示 API 调用次数、Token 用量、预估费用。

**效果：**
- 用户随时了解当前消耗
- 便于决定是否需要切换更便宜的提供商

**实现位置：** `main.py` 的 `add_stats()` + `app.js` 的 `updateStats()`

#### CC-09 费用超限警告

**策略：** 费用超过 5.0 元时，底部状态栏边框变红，提醒用户注意。

**效果：**
- 视觉警告，避免用户无意识地持续消耗
- 阈值可调（当前 `COST_WARN_THRESHOLD = 5.0`）

**实现位置：** `app.js` 的 `updateStats()` 函数

```javascript
const COST_WARN_THRESHOLD = 5.0;
if (usage.estimated_cost >= COST_WARN_THRESHOLD) statsBar.classList.add("warn");
```

#### CC-10 语音识别三级降级

**策略：** DashScope Recognition（云端）→ FunASR（本地免费）→ Google（免费）。

**效果：**
- DashScope 识别质量最高，消耗 API 额度
- FunASR 本地运行，完全免费，无网络依赖
- Google 作为最后兜底，免费但需要网络
- 优先使用云端高质量识别，失败时自动降级到免费方案

**实现位置：** `stt_service.py` 的 `transcribe_audio_sync()` 函数

```python
# 1. DashScope（高质量，消耗额度）
result = _dashscope_realtime_transcribe(audio_bytes, dash_key)
if result["success"]: return result

# 2. FunASR（本地免费）
result = _funasr_transcribe(audio_bytes)
if result["success"]: return result

# 3. Google（免费兜底）
return _google_transcribe(audio_bytes)
```

## 三、成本估算

### 单次调用成本（以通义千问为例）

| 操作 | 输入 Token | 输出 Token | 成本（元） |
|------|-----------|-----------|-----------|
| 纯文字对话（10 字问题） | ~200 | ~100 | ≈ 0.00012 |
| 视觉识别（带图片） | ~800 | ~200 | ≈ 0.00036 |
| 自动观察（单帧变化） | ~800 | ~200 | ≈ 0.00036 |

### 典型使用场景日成本

| 场景 | 日调用次数 | 日成本（元） |
|------|-----------|-------------|
| 轻度使用（偶尔聊天） | 20 次 | ≈ 0.005 |
| 中度使用（带摄像头） | 50 次 | ≈ 0.018 |
| 重度使用（自动观察 1 小时） | 200 次 | ≈ 0.072 |

> 以上基于通义千问 qwen-turbo/qwen-vl-max 定价估算，实际费用以 API 提供商账单为准。
